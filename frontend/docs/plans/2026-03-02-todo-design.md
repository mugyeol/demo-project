# Todo UI Refresh — Modern Minimal Direction

## 1. Context & Intent
The current Todo experience leans into a glassmorphism aesthetic: layered gradients, serif display typography, rounded pills, and pronounced shadows. It feels decorative but not aligned with the request for a modern, minimal tone. The experience should instead emphasize clarity, reduction, and focus—placing the task list and quick input above everything else. Constraints: keep the existing React structure, preserve accessibility attributes (aria labels, live regions, button semantics), and avoid adding stateful logic beyond small presentational tweaks (e.g., icon labels or microcopy). There is no appetite for brand-new flows, so the filters, stats, local-storage persistence, and delete interactions must remain in place. Success is measured by: (a) a calmer visual voice built on restrained color, consistent spacing, and less ornamentation, (b) better hierarchy with a cleaner type system, (c) components that feel native to modern dashboard tooling (think Linear/Height/Arc). Testing should focus on rendering (npm run build) plus a manual sanity check that filters/add/delete still behave because no logic is changing. Since this automation run cannot ask clarifying questions, the spec locks decisions below and doubles down on making them explicit inside the stylesheet comments and documentation.

## 2. Approach Options
**Option A – Framed Surface**: Keep the centered card but neutralize it. Use a single-column shell with generous negative space, subtle frosted border, and a monochrome palette punctuated by one electric accent. Pros: minimal engineering (mostly CSS) and easy to keep responsive. Cons: risk of still feeling decorative if gradients remain heavy. **Option B – Full-Bleed Slate**: Remove the card and let the todo list run full width on a slate background with only dividers. Pros: extremely minimal, matches “system apps”. Cons: loses containment and makes long lists harder to scan on large monitors without additional constraints. **Option C – Panel Stack**: Introduce stacked panels (summary stats, input, list) with micro-grid layout and a tinted background for each. Pros: offers the “app” feel, but introduces more DOM wrappers and grid logic. Given scope, Option A best balances effort and clarity: retain the current structure yet restyle everything—drop the gradients in favor of a desaturated background, switch to Inter/Space Grotesk typography, minimize borders, introduce 1px hairline dividers, and align buttons with modern ghost/filled pairs.

## 3. Design Spec & Implementation Notes
**Palette**: page background `#f5f6f8` with a faint corner accent `#e9ecf5`, card `#ffffff`, text `#101828`, secondary text `#667085`, accent `#111bff`, caution/danger `#d92d20`, divider `#e4e7ec`. **Typography**: import `Space Grotesk` for headings, `Inter` for body/calls to action. **Layout**: `todo-page` remains centered but loses the heavy gradient; `todo-shell` gets a 1px border, 20px radius, and 32px padding on desktop with fluid clamps. **Form/Input**: convert to a slender input bar with left border indicator and a dark accent submit button, featuring a pressed state (translateY). **Filters**: style as segmented controls—use CSS variable for accent to highlight `is-active`, with transparent background otherwise. **Todo items**: flatten cards into minimalist rows: 1px divider, more generous vertical padding, custom checkbox accent color, and hover highlight. Buttons adopt text-style ghost treatment with iconlike appearance while keeping accessible labels. **Feedback states**: `empty` message uses dotted border, small caps text. **Footer**: convert to typographic secondary text with dotted separators via CSS `gap` and `::before`.
