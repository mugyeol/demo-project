# Todo retro tone & manner — 2026-03-04

## Context & Goals
The todo screen already provides a solid feature set (local storage persistence, filtering, bulk clear, stats), so the request focuses entirely on visual tone. “Retro” here means the playful productivity interfaces common in late 70s/early 80s analog planners: warm beiges, burnt oranges, teal accents, rounded corners, offset drop shadows, and grainy backgrounds. Keeping the React tree untouched keeps scope manageable; the work is an App.css rewrite plus a few additional utility wrappers for decorative layers. Primary goals: (1) instantly convey retro vibes without harming readability, (2) preserve existing affordances/focus outlines for accessibility, and (3) ensure the theme still looks intentional on mobile widths. Constraints: single CSS file, no additional dependencies beyond Google Fonts, keep DOM changes minimal, and avoid animations that could trigger motion sickness. Success criteria: backgrounds evoke paper texture using layered gradients, typography mixes a chunky display face with a clean mono body, buttons gain tactile depth via drop shadows, and checkboxes mimic retro rounded squares. The UI must pass WCAG AA for key text elements (ratio ≥ 4.5 on beige surfaces) and degrade gracefully if custom fonts fail.

## Direction Options
**Option A – Warm Stationery.** Palette: parchment (#fff8e1), sienna (#c45b25), teal (#1a8b8f), graphite (#2d1e15). Typography: `"Syne"` for headings (bold, geometric) + `"Space Mono"` for body copy. Adds paper grain overlay and offset card shadows to simulate stacked note cards. Pros: immediate retro association, easy to implement with CSS variables. Cons: mixing display + mono fonts risks readability if sizes not tuned.
**Option B – 90s Neon Terminal.** Dark navy background, green terminal text, pixel fonts. Pros: instantly nostalgic for early PCs. Cons: dark theme breaks current brand brightness, high contrast might be fatiguing, would require rewriting focus styles.
**Option C – Pastel Memphis.** Uses pastel lavender/mint, Memphis squiggles, heavy use of CSS clip-path for shapes. Pros: playful. Cons: busy visuals, significant extra markup for shapes.
Recommendation: Option A balances whimsy with practicality. It keeps the beige card-based layout familiar, requires only CSS updates, and aligns with stationary/planner inspirations that make sense for task tracking.

## Final Experience Design
Typography: import `Syne` (700) for headings and CTAs, `Space Mono` (400/700) for body, lists, and stats to convey retro typewriter vibes. Body font sizes remain ≥ 15px for readability. Color tokens: `--page-bg: #fff5dd`, `--page-bg-2: #fde2b5`, `--surface: #fffaf0`, `--ink: #2f1a12`, `--muted: #6e4b3a`, `--border: #f0d3ae`, `--border-strong: #d3a264`, `--accent: #d75a28`, `--accent-strong: #a63c16`, `--accent-subtle: #ffe1c7`, `--teal: #1a8b8f`, `--danger: #bb1e40`. Buttons get pill shapes with double drop shadows (`0 2px 0 #87300f` + `0 8px 20px rgba(135, 48, 15, 0.35)`). The page background layers a linear gradient with a semi-transparent repeating radial gradient plus `mix-blend-mode:multiply` pseudo-element to mimic paper grain. `.todo-shell` receives a 4px offset border using `::before` to emulate index cards, while `.todo-header` gets accent dividers and retro badges. Checkbox styling: hide native checkbox using `appearance:none`, draw rounded square with accent border, fill with teal checkmark via `linear-gradient`. Empty states use dashed outlines and a retro italic caption. Responsive tweaks: below 560px the card padding decreases, filters stack, and decorative pseudo elements scale down to avoid overlapping content. Motion: single `fadeSlideUp` animation for cards, respects `prefers-reduced-motion`. Focus outlines remain `2px solid var(--teal)` for consistency. EOF
