# Todo tone & manner refresh — 2026-03-03

## Context & Goals
The current todo page leans toward a decorative, glassmorphism-inspired treatment with gradients, saturated greens, and two different typefaces. While visually pleasant, it clashes with the “modern minimal” language the requirement asks for: neutral palettes, crisp typography, restrained ornamentation, and functional spacing. Functionally, the feature set is solid (local persistence, filters, stats), so the redesign focuses on tonality rather than new capabilities. Success looks like (1) a calm, monochromatic surface that still preserves clear hierarchy, (2) components that feel tactile without relying on heavy shadows, and (3) predictable responsive behavior so the UI scales from 360px phones to 1200px desktops without layout jumps. Constraints: keep the existing React structure (no new dependencies), avoid dark backgrounds that could reduce contrast, and maintain accessible touch targets for filters, todo items, and destructive actions. This document defines the new look, ensuring developers can translate it into CSS tweaks without altering business logic. Validation with real users will follow once the automated flow completes, but the spec emphasizes WCAG AA color contrast, focus-visible outlines, and motion reduced to subtle fade scales to respect users prone to motion sensitivity. Overall, the goal is to turn the experience into something that feels like a SaaS productivity suite landing page rather than a marketing splash.

## Direction Options
**Option A – Light Slate Mono:** Inter + Space Grotesk stack, #0f172a ink, cool grey surfaces (#f5f7fb background, #ffffff cards), a single royal blue accent (#2563eb). Buttons become outline pills, the todo list turns into flat cards with 1px dividers, and the background is a simple gradient for depth. Pros: instantly recognizable “modern productivity” vibe, minimal CSS complexity. Cons: somewhat generic, risks blending in with Tailwind defaults.
**Option B – Warm Neutral Minimal:** Use “Satoshi” style geometric sans (substituted with Manrope for availability), ivory background (#f9f7f4), warm charcoal text (#1e1b16), muted amber accents (#f0a500). Pros: distinctive warmth, complements existing fonts. Cons: may still feel decorative and fights requirement for modern coolness.
**Option C – Split Layout Dashboard:** Introduce a dual column layout: left column shows hero/stats, right column contains list. Pros: more “app-like”. Cons: requires more markup changes and risks hurting focus on tasks. Given the lightweight requirement, Option A balances familiarity, focus, and effort, so we will implement it.

## Final Experience Design
Layout: retain single-column shell centered on page, max-width 680px, generous padding. Background becomes `linear-gradient(180deg, #f7f9fc 0%, #eef1f7 100%)` without texture. The page uses `Inter` (400/500/600) via Google Fonts, falling back to `system-ui`. Colors: `--page-bg`, `--surface`, `--border`, `--ink`, `--muted`, `--accent`, `--accent-subtle`, `--danger`. Component specs: header uses compact eyebrow label (#2563eb, uppercase) over 32px/600 `Inter`. Subtitle becomes 15px/1.5 muted text. Form: input spans full width, 14px text, 12px radius, 1.5px border. Include subtle inset shadow on focus and `box-shadow` raising when active. Primary button is 40px tall, 12px radius, high-contrast (#2563eb background, white text). Toolbar: filters become segmented control (grouped buttons with shared border, `is-active` state turns background accent-subtle and text accent). “Clear done” button uses ghost style with iconless label, disabled state uses 40% opacity. Todo items: convert to minimalist cards with `gap: 1rem`, 20px radius, 1px border, `box-shadow: 0 1px 2px rgba(15,23,42,0.08)` for depth. Checkbox replaced via CSS with custom 18px square using accent color; when done, text color shifts to muted and strikethrough, entire card lowers opacity. Empty state uses dashed outline, center text. Footer stats align baseline with uppercase labels. Responsive: at ≤480px, reduce shell padding, stack toolbar. Motion: fade+slide transitions (`translateY(4px)`, `opacity`). Focus-visible for buttons and inputs uses 2px accent outline to guarantee accessibility.
